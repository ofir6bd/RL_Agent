RL OpenKBP — Full Pipeline · V2 per-fraction MDP · symmetric PTV / OAR per-fraction targets · NNLS warm-start before PPO · plain-english walkthrough using the same vocabulary as the lectures

What this project does. It trains a reinforcement-learning (RL) agent to plan a radiotherapy treatment for head & neck cancer patients. Treatment is split into 35 daily sessions ("fractions"). For each session the agent decides how strongly to fire 9 radiation beams from different angles around the patient. The goal is the classic clinical trade-off: deliver enough dose to kill the tumour while sparing the healthy organs next to it.

How it is framed as an RL problem (course vocabulary). We model the problem as a finite-horizon Markov Decision Process $(S, A, p, r, \gamma)$ where each step $t$ is one daily fraction. The agent state $S_t$ is the Markovian summary the network sees (CT + structure masks + cumulative dose so far + remaining gap + $t/35$), the action $A_t$ is the 2,304 beamlet intensities for this fraction, and the scalar reward $R_{t+1}$ scores how well that fraction hit a per-fraction PTV target and stayed under a per-fraction OAR limit. By design we treat each fraction as its own 1-step episode (done = True every step), so the PPO return collapses to $G_t = R_{t+1}$ and credit assignment is local; the cumulative dose carried in the state is what re-introduces the long-term dependencies. We buffer all 35 transitions of a patient and run one PPO update per patient.

Two-stage learning. Before any PPO begins we solve a per-patient non-negative least-squares (NNLS) oracle $a^\ast = \arg\min_{a \geq 0} \lVert \sqrt{W}\,(\text{DIM}\,a - d_\text{gt}/n)\rVert_2^2$ and supervised-pretrain the actor head against $\operatorname{inv\_softplus}(a^\ast)$. This breaks the symmetry of the uniform-init policy — without it, every beamlet for every patient emits the same $\operatorname{softplus}(-1) \approx 0.31$ and PPO has no useful gradient to refine. PPO then takes the warm-started policy and adapts it to maximise the per-fraction PTV/OAR reward.

Coverage status. An earlier release had a bug in the DIM voxel downsampler that left ~25 % of PTV voxels unreachable, capping full-mask D95_PTV* at 0 Gy. The fixed downsampler (Step 3) does source-side aggregation with a target-side nearest-source fallback, so every target voxel now has at least one DIM contributor; PTV reachability is 99.7–100 % across all 206 patients and full-mask D95 matches D95r within rounding.

Glossary — the words you'll keep seeing

MDPmarkov decision process
The mathematical framing for the whole project (Lecture 3). An MDP is the tuple $(S, A, p, r, \gamma)$: a set of states, a set of actions, a transition / reward model $p(r, s' \mid s, a)$, and a discount factor $\gamma \in [0,1]$. Here $S$ is "the patient and what we've already done", $A$ is "the 2,304 beamlet intensities to fire this fraction", and $p$ is fully deterministic (the DIM multiplies the action to get the dose). We assume the Markov property: once the state is known, the history can be thrown away — which is why we explicitly stuff cumulative dose and the remaining gap into the state, so it's a sufficient statistic of the past.
agent / environment state$S_t$ vs the world
Lecture 1's distinction. The environment state is the patient's true biology — hidden and huge. The agent state $S_t$ is the compact, Markovian summary the network actually sees. In our project that's 12 channels of $64^3$ (CT + 8 masks + cum dose + remaining gap + beam paths) plus the scalar $t/35$. Everything the policy/critic decides flows from this $S_t$.
reward $R_{t+1}$ & return $G_t$scalar feedback + cumulative
The reward hypothesis (Lecture 1): any goal can be expressed as maximisation of a scalar reward. After every fraction the environment emits one number $R_{t+1}$ — our PTV bonus minus the OAR penalty. The return is the (discounted) cumulative reward $G_t = R_{t+1} + \gamma R_{t+2} + \ldots$ that the value function tries to estimate. In V2 each fraction is a 1-step episode so $G_t$ simplifies to just $R_{t+1}$.
policy $\pi$ & value $v_\pi$actor & critic
A policy $\pi(a \mid s)$ maps state to a distribution over actions — in our code, the actor head that outputs a Gaussian over 2,304 raw beamlet weights (then softplus). The value function $v_\pi(s) = \mathbb{E}[G_t \mid S_t = s, \pi]$ is the expected return from $s$ under $\pi$ — in our code, the critic head. The advantage $A_\pi(s, a) = q_\pi(s, a) - v_\pi(s)$ is what PPO's gradient is weighted by; we estimate it with GAE.
episodefinite-horizon trajectory
A sequence of $(S, A, R)$ tuples that ends when done = True. In V1 an episode was the full 35-fraction patient course (long credit-assignment chain). In V2 an episode is a single fraction — we set done = True after every step. A patient is therefore 35 consecutive 1-step episodes that share the same persistent cumulative_dose. We collect all 35 into one buffer (the "rollout") and run one PPO update per patient.
OARorgans at risk
Healthy organs near the tumour that we want to protect. In this project there are exactly five: brainstem, spinal cord, mandible (jaw bone), left parotid (saliva gland) and right parotid. Each has a "safe limit" in Gy that we don't want the average dose to cross.
PTVplanning target volume
The tumour region the plan must "cover" with a prescribed dose. There are three classes — PTV70 needs 70 Gy, PTV63 needs 63 Gy, PTV56 needs 56 Gy — spread across the 35 sessions. Most head & neck patients have all three nested volumes (PTV70 is the gross tumour, PTV63 is high-risk margin, PTV56 is elective nodes).
Gygray (the unit of dose)
1 gray = 1 joule of radiation absorbed per kilogram of tissue. Tumours need 56–70 Gy total; spinal cord must stay under 45 Gy. That's why every number in this project is in Gy.
fractionone treatment session
One day's session. There are 35 of them in a full course. Each session adds a small amount of dose (~2 Gy/session for PTV70) so the body has time to recover between sessions.
beam & beamletradiation pencils
A "beam" comes from one angle around the patient (we use 9 angles spaced 40° apart). Each beam is split into a grid of tiny pencil beams called "beamlets" (16 × 16 = 256 per beam). The agent controls each beamlet's intensity independently — that's how you sculpt the dose around an organ.
DIMdose-influence matrix
A pre-computed table that says: "if I fire beamlet j at intensity 1, voxel i receives X Gy." It has 262,144 rows (one per voxel) and 2,304 columns (one per beamlet). Once it's built, computing a dose plan is just one matrix multiplication — milliseconds instead of minutes.
CT & HUthe anatomy image
CT is a 3D X-ray scan of the patient. Brightness in standard radiology is measured in Hounsfield Units (HU): air = −1000, water = 0, bone ≈ +1000. OpenKBP stores CT as HU + 1000 (a non-negative integer), so the CSVs you'll see contain values from ~0 (air) up to ~4000 (dense bone / metal), with body tissue clustering around 1000. Before feeding it to the network we clip to a fixed window and rescale into [−1, 1] so it sits on the same scale as the binary masks.
voxel3D pixel
A 3D pixel. Each voxel has a position in the patient's body and one number per layer (CT brightness, dose, mask value, …). The grid in this project is 64 × 64 × 64 = 262,144 voxels.
D95tumour coverage
The dose received by at least 95 % of the tumour. The clinical gold standard: if D95 ≥ prescription, the tumour is well covered. If D95 falls below, the coldest 5 % of the tumour may not get enough dose.
DVHdose-volume histogram
A curve summarising the whole dose distribution in one shape. The OpenKBP challenge scores plans by comparing their DVHs to the clinical ones — lower difference = better plan.
MAEmean absolute error
Average of |predicted dose − clinical dose|, computed only over voxels inside the body. Tells you how close the agent's plan is to the human dosimetrist's plan on average.
PPOproximal policy optimisation
The on-policy policy-gradient algorithm doing the actual learning. It updates a stochastic policy $\pi_\theta(a\mid s)$ to maximise expected return, but each update is clipped to a trust region (probability ratio kept in $[1-\varepsilon, 1+\varepsilon]$ with $\varepsilon = 0.2$) so the new policy never moves too far from the old one in a single step.

When does it update in this project? Each fraction is a 1-step episode, but we don't update the network after every fraction — that would mean a batch of 1 and the math (advantage normalisation, mini-batches) falls apart. Instead we buffer all 35 fractions of one patient, then run a single PPO update on those 35 transitions, then move on to the next patient. So "1 patient = 1 PPO update = 35 buffered 1-step episodes".
GAEgeneralised advantage estimation
PPO's estimator of the advantage $A_t = q_\pi(s,a) - v_\pi(s)$ — "was this action better or worse than the critic expected from this state?" In the general case GAE blends every $k$-step TD residual $\delta_t = R_{t+1} + \gamma\,v(S_{t+1}) - v(S_t)$ with an exponentially decaying weight $\lambda$:
A_t = δ_t + (γλ)·δ_{t+1} + (γλ)²·δ_{t+2} + …

In this project the formula collapses. Because every transition has done = True (1-step episodes), the bootstrap term $\gamma\,v(S_{t+1})$ is masked out and the sum truncates after the very first term, giving simply
A_t = R_{t+1} − V(S_t)  &  return_t = R_{t+1}.

So $\gamma = 0.995$ and $\lambda = 0.95$ are kept in the config for compatibility but they no longer affect the advantage in V2 — the long-term dependencies live entirely in the Markovian state (cumulative dose, remaining gap, $t/35$), not in a discounted return.
softplusthe "never negative" function
softplus(x) = log(1 + eˣ). Always ≥ 0, can get very close to zero. We apply it to the network's raw output because you cannot deliver negative radiation, but we still want the agent to be able to "turn a beamlet almost off".
NNLS warm-startper-patient oracle plan $a^\ast$
Before PPO starts, we solve a weighted non-negative least-squares problem per patient: $a^\ast = \arg\min_{a \geq 0}\,\lVert\sqrt{W}(\text{DIM}\,a - d_\text{gt}/n_\text{fractions})\rVert^2$ via FISTA. $W$ upweights PTV voxels (×50) and OAR voxels (×5) so PTV coverage dominates the fit. Result: a per-beamlet plan that, if repeated identically every fraction, approximates the clinician's full dose. We cache $a^\ast$ as warmstart_action.npy and use it as a supervised target for the actor head (Step 4a). Acts as both an imitation prior and a practical upper bound on what PPO can achieve given the current DIM.
reachabilityDIM coverage check
A voxel $v$ is reachable if at least one beamlet has a non-zero DIM entry into it (row-sum > 0). The original downsampler did integer-floor source-to-target indexing, which left ~25 % of PTV voxels unreachable whenever pyRadPlan's native dose grid was coarser than 64 in any axis (typical z-axis: 47 slices → 64 targets). Full-mask D95 was therefore mathematically capped at 0 Gy regardless of the agent. The fix in _downsample_matrix uses source-side aggregation plus a target-side nearest-source fallback so every target voxel has at least one DIM contributor; current PTV reachability is 99.7–100 % across all 206 patients. evaluate.py still reports D95r and %reach so any future regression of this guarantee shows up immediately.
splittrain / validation / test
Three groups of patients. Train is what the agent learns from. Validation is held out so we can compare runs and pick the best checkpoint. Test is touched once at the very end for the final, unbiased number.

Step 1 · Clean, resample, and stack into 3D arrays
load_patient → SimpleITK resample to 64³ → CT → [−1, 1] → ray-cast beam paths

This step turns the sparse CSVs into clean, ready-to-use 3D arrays.

For every patient we (1) rebuild each organ mask and dose volume back into a full 3D grid, (2) resample everything to a fixed 64 × 64 × 64 cube using the real millimetre spacing so patients of different scan depths all end up the same shape, (3) subtract 1000 from the CT (OpenKBP stores it as HU + 1000, so disk values run ~0 to ~4000) to recover true Hounsfield units, then clip to [−1000, +1000] HU and divide by 1000 → [−1, 1], and (4) pre-draw a simple "beam-path map" — for each of the 9 gantry angles (spaced 40° apart) we cast one ray through the iso-centre and extrude the touched (x, y) voxels along the full z range.

If an organ file is missing for a patient (not every patient has every organ), it becomes an all-zero mask channel and the reward and metrics layers know to skip it — nothing crashes.

Step 3 · Build the Dose Influence Matrix (DIM) slow, but only once
pyRadPlan(SVDPB) · 9 gantry angles 40° apart · 16 × 16 beamlets per beam · → dose_influence_matrix.npz of shape (262144 × 2304)

pyRadPlan is an open-source radiotherapy physics library. We hand it the patient's CT, the organ outlines, and our beam setup (9 angles, each split into a 16 × 16 grid of beamlets = 256 beamlets/beam = 2,304 beamlets total). It then simulates how X-rays travel through the body — taking into account that bone, muscle and air absorb differently.

The result is the Dose Influence Matrix (DIM): a big table with one row per voxel (262,144 rows) and one column per beamlet (2,304 columns). Each cell answers: "how many Gy does this beamlet deposit in this voxel at intensity 1?"

This simulation is slow (5–10 min per patient), but it only runs once per patient — the body doesn't change between sessions. The huge payoff: during training, computing the dose for any choice of beam intensities becomes one fast matrix multiplication: dose = DIM · intensities.

Voxel downsampler (now fixed). pyRadPlan returns the DIM on its own dose grid (typical: 47×100×100, often coarser than 64 in z) and we resample it to 64³ with a sparse (64³, V_dose) mean-pool matrix. The original implementation routed every source voxel into floor(s·grid/N), which for axes with N<grid left whole rows of target voxels unmapped — leaving ~25 % of PTV voxels with all-zero DIM rows and pinning full-mask D95 to 0 Gy. The fix is two-pass: (1) every source voxel still aggregates into its target via the nearest-centre rule floor((s+0.5)·grid/N) (correct mean-pool when downsampling); (2) any target voxel left unfilled by step 1 (only happens when N<grid) gets a single nearest-source contribution floor((t+0.5)·N/grid). Row-totals are then normalised so the matrix stays row-stochastic. After the fix every target voxel is covered and PTV reachability is 99.7–100 % on all 206 patients.

Step 3b · Per-patient NNLS warm-start oracle plan a*
weighted FISTA-NNLS · target = $d_\text{gt}/n_\text{fractions}$ · weights: PTV ×50, OAR ×5, body ×1 · → warmstart_action.npy of shape (2304,)

This step pre-computes a high-quality beamlet plan per patient that the actor head will imitate before PPO begins. Without it, the actor head starts from a uniform initialisation and PPO cannot move it in any reasonable number of episodes (the policy stays frozen at softplus(-1) ≈ 0.31 per beamlet).

We frame it as weighted non-negative least-squares:

$a^\ast \;=\; \arg\min_{a \geq 0}\;\lVert \sqrt{W}\,(\text{DIM}\,a \;-\; d_\text{gt}/n_\text{fractions}) \rVert_2^2$

  • Target $d_\text{gt}/n_\text{fractions}$ — the per-fraction "fair share" of the clinician's total dose. If the agent could repeat $a^\ast$ every fraction, the cumulative would approximate dose_gt.
  • Per-voxel weights $W$ — body voxels weight 1, OAR voxels weight 5, PTV voxels weight 50. Without weighting, body voxels (~95 % of the volume) drown the fit and PTVs are under-dosed.
  • Solver — FISTA (accelerated projected gradient): each iteration does $a \leftarrow \max(0,\, y - \eta\,\text{DIM}^\top\!(W \odot (\text{DIM}\,y - b)))$ with Nesterov momentum, step size $\eta = 1/L$ where $L$ is estimated via 20 rounds of power iteration on $\text{DIM}^\top \operatorname{diag}(W)\,\text{DIM}$. ~400 iterations per patient, ~10 s on CPU.

Cache: one warmstart_action.npy per patient. Typical $a^\ast$ is sparse — only ~1000 of 2304 beamlets are above $10^{-3}$ — which is exactly the kind of structured plan that beats the uniform-init baseline immediately.

see scripts/compute_warmstart_actions.py · solver fista_nnls_weighted
Phase 2 — Training loop (one patient = 35 one-step episodes = 1 PPO update)
Step 4a · Supervised actor pretraining on $a^\ast$ runs once, before PPO
target = $\operatorname{inv\_softplus}(a^\ast / \text{ACTION\_SCALE})$ · loss = MSE on actor_mu output · 5 epochs · minibatch 4 · Adam lr 3e-4 · updates encoder + actor trunk + actor_mu

Before PPO starts, we load each training patient's $a^\ast$ from Step 3b, build the fraction-0 state, and fit the actor head so that its deterministic output reproduces $a^\ast$. Concretely we regress actor_mu(features(state, t/35=0)) against $\operatorname{inv\_softplus}(a^\ast) = \log(\exp(a^\ast) - 1)$, floored at $-8$ so zero-beamlets don't blow up to $-\infty$.

Why pretrain $\mu$ instead of the full distribution? We want to keep the on-policy exploration noise (log_std) intact for PPO — we only need to break the per-beamlet symmetry of the uniform initialisation. The critic and log_std are deliberately left at their init values; PPO learns them from scratch.

Result. Post-pretrain the deterministic policy already produces ~80 % active beamlets with diverse intensities (mean 0.03, max 0.7) per patient, and the very first PPO episode scores a positive reward instead of the strongly-negative reward of a uniform-init agent.

We save the post-pretrain weights to runs/warmstart.pt so you can evaluate the imitation policy alone and compare it to the post-PPO policy.

Step 4 · Start a new patient course env.reset()
pick a train patient → load .npy + dose_influence_matrix.npz → cumulative_dose = zeros(64³) → fraction_index = 0

A patient course means treating one patient through all 35 fractions. env.reset() is only called once per patient — not between fractions — so the cumulative dose, fraction counter and Markov state all carry across the 35 in-course steps. Before each course we pick a random patient from the train split, load their cached arrays and DIM into memory, zero out the running cumulative dose volume (the patient hasn't received anything yet), and reset the session counter t = 0.

Each step() inside the course is its own 1-step episode (done = True), but the environment's internal state — cumulative_dose, fraction_index, self._patient — persists, so the next call's $S_{t+1}$ is the Markovian continuation, not a fresh start. The training loop knows when the course is over by checking info["patient_done"].

Step 5 · Build the state — what the agent sees
12 channels of 64 × 64 × 64 + a scalar t/35

Before deciding anything, the agent looks at a stack of 12 3D layers, each covering the whole patient:

ch 0CT scan (anatomy)The patient's tissue map in [−1, 1] — lets the agent see where bone, soft tissue and air are so it knows what each beam will be passing through.
ch 1–3PTV70 · PTV63 · PTV56 masksThree binary 0/1 volumes pinpointing exactly where the three tumour targets sit and what dose level each one is asking for.
ch 4–8Brainstem · SpinalCord · Mandible · L/R Parotid masksFive binary masks marking the organs the agent must protect, each on its own channel so the network can learn organ-specific avoidance behaviour.
ch 9cumulative dose so far ÷ 70A running tally of the dose already delivered in earlier sessions — this is the agent's memory of "what have I done up to now".
ch 10remaining tumour dose ("the gap") ÷ 70Per-voxel difference between each PTV's prescription and the dose accumulated so far — a live to-do map showing which tumour regions still need more radiation.
ch 11beam-path mapA pre-drawn sketch of which voxels each of the 9 gantry angles passes through, so the agent understands the geometric reach of every beam direction without having to learn it from scratch. All 9 angles are merged into this one volume — the channel does not distinguish which beam touched which voxel, only "is this voxel inside some beam path."

Why divide by 70? Networks learn best when all inputs sit in a similar small range. Mixing huge numbers (50 Gy) with small ones (binary masks of 0/1) confuses them.

A single extra number — t/35, "what fraction of the course is done" — is not stacked as a channel. It's appended later, after the CNN compresses the 12 layers.

Step 6 · 3D-CNN encoder — squeeze the volume into 128 numbers
12 × 64³ → Conv3d × 4 (stride 2) → 4³ → AvgPool → 128 → +1 scalar t/35 → 129 features

The 12 × 64³ state has ~3 million numbers — far too many for the decision-making layer. A 3D Convolutional Neural Network (CNN) slides small 3D filters over the whole volume, looking for meaningful local patterns (e.g. "tumour voxel right next to spinal cord that's already nearly at its limit").

Four convolution layers halve the volume each time (64 → 32 → 16 → 8 → 4) while doubling the number of "feature channels" (12 → 16 → 32 → 64 → 128). A global average-pool collapses the 4³ × 128 feature volume into 128 numbers, and a small linear layer projects them into the final 128-number summary vector.

Finally we tack on the single scalar t/35, giving a 129-number feature vector. Both the actor (which picks an action) and the critic (which scores the state) read from this same vector.

Step 7 · Actor head — the agent picks its action
features (129) → Linear 256 + ReLU → Linear 256 + ReLU → mu (2,304) ; std = clamp(log_std, [−5, 1]).exp(); raw ~ Normal(mu, std); action = softplus(raw) · ACTION_SCALE

This is the heart of the RL agent. The 129-number feature vector from Step 6 is fed into the actor trunk: two fully-connected layers of width 256, each followed by ReLU. The output of the trunk goes through one more linear layer — actor_mu — which produces a vector of length 2,304, one number per beamlet (9 beams × 16 × 16 = 2,304). That vector is mu: the centre (mean) of the Gaussian the agent will sample from for each beamlet. It is not the physical intensity yet — mu lives in raw pre-softplus space and can be negative; only after the sample is passed through softplus (see below) does it become the non-negative intensity vector that gets multiplied by the DIM in Step 8.

Initialisation matters for this problem. The actor_mu weights use orthogonal init with gain 0.01 and the bias is initialised to ACTOR_MU_BIAS_INIT = -1.0, so a fresh untrained policy outputs $\mu \approx -1$ for every beamlet — and therefore the deterministic action is $\operatorname{softplus}(-1) \approx 0.31$ uniformly. That uniform-init policy is a pathological local minimum: with every beamlet producing the same intensity, PPO's per-beamlet gradient is dominated by noise and the policy stays frozen at init for hundreds of episodes. This is why Step 4a's supervised warm-start exists — it replaces this uniform $\mu$ with the per-beamlet $\operatorname{inv\_softplus}(a^\ast)$ before PPO ever runs.

Adding randomness for exploration. The agent doesn't act on mu alone — it builds a Normal (Gaussian) distribution around it. The standard deviation isn't predicted from the state; it's a state-independent, learnable parameter called log_std, one value per beamlet, initialised to ACTOR_LOG_STD_INIT = -1.0 (so the initial std is exp(−1) ≈ 0.37). At every forward pass log_std is clamped to $[-5, +1]$ so std stays finite and not too tiny (which kills exploration) or too large (which makes the noise dwarf the mean). The actual std used at every step is std = log_std.clamp(-5, 1).exp(), broadcast to match mu's shape. Storing it as a log keeps it strictly positive no matter what gradient PPO pushes through it.

Sample vs. mean — train vs. eval. The agent's act() method has a deterministic flag:

  • Training (deterministic=False): raw = dist.sample() — one random draw from Normal(mu, std) per beamlet. This is what lets the agent explore intensities it has never tried.
  • Evaluation (deterministic=True): raw = dist.mean (which is just mu). No randomness — running the same patient twice produces the exact same plan, which is mandatory for clinical reproducibility.

Log-probability for PPO. Whatever raw ends up being, the actor also returns logp = dist.log_prob(raw).sum(-1) — the total log-probability of that sample under the current policy. PPO stores this old log-prob alongside the action and later compares it to the new log-prob after an update, forming the clipped probability ratio that keeps the policy from changing too aggressively.

From raw sample to physical intensities. A Normal is unbounded — raw can be negative, but you can't deliver negative radiation. So we run it through softplus (F.softplus(raw)) and scale by ACTION_SCALE = 1.0: a smooth function that maps any real number to a strictly positive one, while still letting the output get arbitrarily close to zero (≈ "turn this beamlet almost off"). The result is the action — the 2,304-long vector of non-negative intensities that gets shipped to Step 8 and multiplied by the DIM.

Important asymmetry to remember. The Normal distribution lives in raw space, but the environment only ever sees the post-softplus action. PPO computes log_prob on the raw sample (not the softplus output) — that's the design choice in this codebase and it works because softplus is monotonic, so improving raw reliably moves the action in the same direction.

Step 8 · Apply the DIM — turn intensities into dose
fraction_dose = DIM · intensities ; mask outside body ; cumulative_dose += fraction_dose

Now we cash in the DIM we built earlier. The 2,304-long intensity vector is multiplied by the DIM, instantly giving the dose (in Gy) that each of the 262,144 voxels would receive in this single session.

We then multiply by the possible-dose mask — OpenKBP's region where dose is allowed to be evaluated (essentially the body and a thin margin) — so any spillover into empty air is zeroed out. The result is then added to the running cumulative dose volume.

That cumulative dose is what the next state shows the agent through channels 9 (how much has been delivered) and 10 (how much is still missing in the tumour) — so the agent always knows where it stands.

Step 9 · Reward signal per-fraction, two symmetric terms
$R_{t+1} \;=\; \lambda_{\text{ptv}}\cdot\text{ptv\_reward}(\text{fraction\_dose, prescription\_volume, cumulative\_dose\_before, fractions\_remaining}) \;-\; \lambda_{\text{oar}}\cdot\text{oar\_penalty}(\text{fraction\_dose, oar\_tolerance\_total}/n)$
The agent is never told "the correct beam intensities" — those don't exist in the data. Instead, after every fraction the environment emits one scalar reward $R_{t+1}$ that says how good this fraction's dose was, given the cumulative dose already delivered. That is the only learning signal: the reward hypothesis in action.

In V2 both terms are per-fraction and symmetric: the PTV term scores how close this fraction's dose came to its fair share of the remaining tumour gap, and the OAR term penalises this fraction's dose exceeding a per-fraction tolerance share. No terminal bonus, no shaping that depends on the final session — every fraction from #1 to #35 carries an equally dense signal, which is exactly what makes the 1-step-episode framing work.
  • 1 · PTV reward — hit your fair share of the remaining gapevery fraction · adaptive target
    prescription_volume = PTV_mask × prescription_gy
    Per-voxel prescription map. For each PTV (PTV70, PTV63, PTV56) take its binary mask and multiply by its required Gy value. The result is a 3D volume where every tumour voxel carries its own target dose and everything outside is 0.
    per_fraction_target_dose_v = max(prescription_volume_v − cumulative_dose_before_v, 0) / fractions_remaining
    Fair-share target for this fraction. For every PTV voxel $v$: take what's still needed to reach the prescription (max(·, 0) clips voxels that are already at or above), then divide by the number of fractions remaining (including this one). This is the adaptive trick: if an earlier fraction under-dosed, fractions_remaining is smaller and the target per voxel grows — the agent is told to compensate, not punished forever for past slip-ups.
    relative_error_v = | fraction_dose_v − per_fraction_target_dose_v | / per_fraction_target_dose_v   (over voxels with per_fraction_target_dose_v > 0)
    Relative error per voxel. How far this fraction's actual dose was from the per-voxel fair share, expressed as a fraction of that share. Voxels with per_fraction_target_dose_v = 0 (already saturated) are excluded — they contribute nothing here, which keeps the term focused on remaining work.
    ptv_reward = 1 − clip( mean(relative_error), 0, mean_error_clip = 2 )
    Final PTV reward. Average the relative error across active PTV voxels and flip the sign so "perfect" is a positive number. Range: $[-1, +1]$. +1 = delivered exactly the fair share to every voxel; 0 = average error of 100 % (e.g. delivered nothing while expected ~prescription/35); −1 = average error of ≥200 % (badly over- or under-shot). The env multiplies by $+\lambda_{\text{ptv}} = 1.0$.
    Symmetric counterpart to the OAR term: just as the OAR limit is "don't exceed tol/35 in any fraction", the PTV target is "deliver roughly your fair share of the remaining gap this fraction". Adaptive targeting means the agent can recover from past mistakes — if it under-dosed at $t = 5$, the per-voxel target at $t = 6$ is slightly larger and a successful catch-up still scores near +1.
    Example. A PTV70 voxel needs 70 Gy total. At fraction 1 the agent should deliver $\approx 70/35 = 2.0$ Gy — hitting 2.0 Gy gives err = 0, reward 1.0. Suppose it only delivers 1.5 Gy at $t=1$ (err = 25 %). At $t = 2$ the remaining gap is 68.5 Gy over 34 fractions, so target is $\approx 2.015$ Gy. Hit that and you still score 1.0 — the deficit was absorbed by the adaptive target.
    Why a per-fraction PTV target instead of one terminal coverage bonus? Because every fraction now needs its own scalar reward (1-step episodes), and a sparse terminal-only signal would be useless for 34 of the 35 steps. Why clip mean(err) at 2? So a catastrophic random first fraction can't drag the reward below $-1$ and destabilise the value estimate.
  • 2 · OAR penalty — don't exceed your per-fraction tolerance shareevery fraction · per-voxel + mean + Dmax
    per_fraction_oar_tolerance = oar_tolerance_total / n_fractions
    Per-fraction tolerance share. Each OAR has a clinical full-course limit (Brainstem 54 Gy, Cord 45 Gy, Parotid 26 Gy, …). Divide by $n = 35$ to get the per-fraction budget (Brainstem $\approx 1.54$, Cord $\approx 1.29$, Parotid $\approx 0.74$). Mathematical guarantee: if every fraction stays under per_fraction_oar_tolerance, the cumulative is provably under oar_tolerance_total by the end of the course — no late-course blow-up possible.
    voxel_excess_v = max(0, (fraction_dose_v − per_fraction_oar_tolerance) / per_fraction_oar_tolerance)
    Per-voxel excess. How far this fraction's dose to voxel $v$ exceeded the per-fraction limit, as a fraction of that limit. voxel_excess_v = 0 = within budget; voxel_excess_v = 0.20 = 20 % over. Identical formula to V1, just applied to fraction_dose vs oar_tolerance_total/35 instead of cumulative dose vs oar_tolerance_total.
    piecewise_excess_penalty(e) = e  if e ≤ 0.10  else e²  (saturated at e = 5)
    Piecewise penalty shape. Small overshoots get a linear penalty (stable gradient); above 10 % they grow quadratically; above 500 % they saturate so a single wild beamlet during early random exploration can't blow up the critic with NaN-sized targets.
    per_voxel_penalty = mean over OAR voxels of piecewise_excess_penalty(voxel_excess)
    Per-voxel term — main signal. Average $\text{piecewise\_excess\_penalty}(\text{voxel\_excess})$ over every voxel of this organ. Real radiation damage is local: a single hot voxel matters even if the rest of the organ is cold.
    mean_dose_penalty  = piecewise_excess_penalty( excess of organ-mean fraction_dose )
    Mean term — stabiliser. A smooth, low-variance signal that already moves slightly before any voxel crosses the per-fraction limit. Sub-weight oar_mean_subweight = 0.2.
    max_dose_penalty = piecewise_excess_penalty( excess of organ-max  fraction_dose )  (serial OARs only)
    Dmax term — hot-spot guard. Only fires for organs flagged organ_is_serial = true in the config (Brainstem, SpinalCord), where one over-dosed nerve voxel can disable the whole bundle.
    oar_penalty = Σ_organs organ_weight · ( oar_voxel_subweight·per_voxel_penalty + oar_mean_subweight·mean_dose_penalty + oar_dmax_subweight·max_dose_penalty )
    Total OAR penalty. Linear combination of the three terms (sub-weights $\text{oar\_voxel\_subweight}=1$, $\text{oar\_mean\_subweight}=0.2$, $\text{oar\_dmax\_subweight}=1$), scaled by an optional per-organ organ_weight (all $1.0$ by default), summed across organs. The env multiplies by $-\lambda_{\text{oar}} = -1.0$ to flip it into a negative reward contribution.
    Same per-voxel + mean + Dmax mix as V1, but with two crucial changes: (a) evaluated on fraction_dose, not cumulative_dose — so the signal is dense from fraction 1; and (b) compared against per_fraction_oar_tolerance = oar_tolerance_total / n_fractions, not the full-course tolerance — symmetric with the PTV reward and with a built-in cumulative safety guarantee.
    Example — cord. Per-fraction limit $\approx 1.29$ Gy. A safe fraction that puts 0.8 Gy on every cord voxel scores 0 (no penalty). A reckless fraction that hits 3 Gy on one cord voxel gets voxel_excess = 1.32, $\text{piecewise\_excess\_penalty}(\text{voxel\_excess}) \approx 1.74$ — small per_voxel_penalty contribution, plus a hefty max_dose_penalty term because cord has organ_is_serial = true.
    Why per-fraction limits instead of cumulative vs total? A cumulative penalty stays at zero for most of the course (cumulative_dose only approaches oar_tolerance_total near $t = 35$), giving PPO no gradient on early fractions — exactly where it most needs to learn to be careful. The per-fraction formulation fires every fraction the agent is too aggressive. Trade-off: it's stricter than clinical reality (which only cares about the final cumulative), so the agent can't "over-dose now, under-dose later to balance" — we accept that conservativeness for the cleaner signal.
R_{t+1} = +λ_ptv · ptv_reward(fraction_dose, prescription_volume, cumulative_dose_before, fractions_remaining)
       − λ_oar · oar_penalty(fraction_dose, oar_tolerance_total / n_fractions)

λ_ptv = 1.0   ·   λ_oar = 1.0   (set in configs/default.yaml)   ·   no terminal bonus in V2
The two opposing signs encode the clinical trade-off: the PTV term pulls actions up (deliver dose to tumour), the OAR term pulls them down (don't fry healthy tissue). The best plan maximises one while staying inside the other's per-fraction budget — and because both are evaluated per fraction, the agent learns the trade-off from step 1 instead of waiting for a terminal grade.

Step 10 · PPO update — one update per patient src/agents/ppo.py
35 one-step episodes in the buffer · advantage collapses to $R_{t+1} - V(S_t)$ · clip $\varepsilon = 0.2$ · 4 epochs × minibatch 64 · lr 3e-4 · grad clip 0.5 · rewards & values defensively clipped to $[-50, +50]$ · NaN-skip

Once the patient course is over, the rollout buffer holds 35 transitions $\{(S_t, A_t, R_{t+1}, V(S_t), \log\pi(A_t \mid S_t), \text{done}=\text{True})\}$. We then run a single PPO update on that buffer.

(0) Defensive clipping. Before any maths runs we clip both rewards and values to $[-50, +50]$. The OAR penalty is piecewise-quadratic and a single very bad fraction during early exploration can otherwise produce a $\sim 10^3$-magnitude reward that destabilises both the value target and the advantage normalisation. Anything that comes out non-finite (NaN or ±∞) after this clipping causes the entire update to be skipped — better to lose one patient's gradient than to poison Adam's running moments with NaNs that infect every future step.

(a) Compute advantages — GAE collapses. The critic predicted $V(S_t)$ for each fraction; the environment told us the actual reward $R_{t+1}$. The full GAE formula sums TD residuals $\delta_t = R_{t+1} + \gamma\,V(S_{t+1}) - V(S_t)$ with decay $(\gamma\lambda)$ — but because every transition has done = True, the bootstrap $\gamma\,V(S_{t+1})$ is masked out and the geometric sum truncates after the very first term:
    A_t = R_{t+1} − V(S_t)  &  return_t = R_{t+1}.
So the critic is effectively a per-fraction baseline (state-dependent expected reward) and the policy gradient is $\nabla \log\pi(A_t \mid S_t) \cdot (R_{t+1} - V(S_t))$. Advantages are normalised to mean 0 / std 1 across the 35 transitions for variance reduction.

(b) Clipped policy update. Standard PPO surrogate: probability ratio $\rho_t = \pi_{\theta_{\text{new}}}(A_t \mid S_t) / \pi_{\theta_{\text{old}}}(A_t \mid S_t)$ is clipped to $[1-\varepsilon, 1+\varepsilon] = [0.8, 1.2]$ and the actor loss is $-\mathbb{E}[\min(\rho_t A_t, \,\text{clip}(\rho_t)\,A_t)]$. The clipping enforces a trust region so the new policy can't drift too far from the old one in a single update.

(c) Critic update. Clipped value loss (PPO2 style): $V_\theta(S_t)$ is trained against the return target with an additional clip $V_{\text{old}}(S_t) \pm \varepsilon$ that limits how far the critic can move per step — in our 1-step-episode setting the return target is simply $R_{t+1}$, so the critic is just regressing the per-fraction reward conditioned on state. After enough updates $V(S_t)$ becomes the expected per-fraction reward from $S_t$ — the baseline that powers the advantage in (a).

(d) Entropy bonus. Small reward (coef $0.01$) for the policy distribution staying spread out, so the agent keeps exploring different beamlet intensities rather than collapsing onto one rigid plan too early — the classic exploration-exploitation knob from Lecture 2.

All of this is repeated for 4 epochs over the 35-transition rollout in mini-batches of 64 (so effectively one full-batch pass per epoch given $T = 35 < 64$). Gradients are clipped to L2-norm 0.5 and Adam steps with lr = 3e-4. Per-batch NaN-guards re-check every parameter's gradient before opt.step() — any non-finite gradient skips that batch's update entirely.

Why one update per patient instead of one per fraction? A batch of 1 transition makes advantage standardisation meaningless and the PPO clipping math degenerates. A batch of 35 is small but stable, and we still get $\approx 35\times$ more updates per epoch than the V1 design (which only updated once per 35-step rollout with long-horizon credit assignment).

Checkpoints. Three files are written under runs/:

  • warmstart.pt — saved once right after Step 4a, before any PPO update. Lets you evaluate the imitation policy alone.
  • best.pt — saved whenever the rolling mean of the last best_rolling_window = 10 patients' total rewards sets a new max. The previous "single-patient max" rule used to pin best.pt to whichever training patient happened to be easiest, because patients have wildly different reward scales — the rolling mean fixes that.
  • ep<N>.pt every eval_every = 50 patients, and last.pt at the end of training so a final checkpoint always exists regardless of eval_every.

Live diagnostics in the tqdm bar. Each patient also prints R (this patient's total reward), Ravg (rolling mean), pi / v (policy & value losses), mu (mean of actor_mu.bias), lstd (mean log_std), a_mean / a_max (action stats from the last fraction). A frozen policy — the bug we hit before the warm-start was added — is then immediately visible: mu stays at $-1.000$, a_mean stays at $\approx 0.31$, and reward never moves. After the warm-start fix those numbers move every episode.

Phase 3 — Evaluation (deterministic, no learning)
Step 11 · evaluate.py — grade the trained agent deterministic=True
runs on cfg.eval_split (validation by default) · pass --split test for the final report

Evaluation has the same loop as training — 35 sessions per patient — with two important changes:

(1) Deterministic actions. No random sampling; the agent always uses the policy's mean. This makes plans reproducible (run twice → same plan), which is essential for a real clinical workflow.

(2) No learning. The network weights are frozen. We're measuring, not training.

For each patient we then compute:

  • MAE — average |predicted − clinical| dose, masked to the possible-dose region
  • DVH score (proxy) — mean over present structures of |ΔD95| + |ΔDmean| vs ground truth (lower is better)
  • D95 per PTV (full mask) — coverage of each tumour class. Now that the DIM downsampler covers every target voxel, this is the headline coverage metric again. With the current 200-patient PPO run the validation MEAN is D95_PTV70 ≈ 48.4 Gy (target 70), D95_PTV63 ≈ 44.6 Gy, D95_PTV56 ≈ 43.1 Gy.
  • D95r per PTV (reachable-only) — D95 computed over the subset of PTV voxels with at least one non-zero DIM entry. After the downsampler fix this matches the full-mask D95 within rounding for every patient (reachability ≥ 99.5 %); the column is kept as a guardrail in case a future DIM change re-introduces gaps.
  • %reach per PTV — fraction of PTV voxels that are physically reachable by any beamlet. Currently 99.7–100 % across all 206 patients; was 72–76 % before the downsampler fix.
  • Mean dose per OAR — for each healthy organ

Results are printed as one aligned table, with a final MEAN row across patients. Structures that don't exist for a given patient show as - instead of misleading zeros.

Which split to use? The default validation split is what you use while iterating — tweak rewards or hyper-parameters, re-evaluate, repeat. The test split is run once, at the very end, for the final unbiased number to report.

Which checkpoint to use? runs/warmstart.pt is the imitation-only policy from Step 4a; runs/best.pt is the rolling-mean best during PPO; runs/last.pt is the final policy. Comparing them is the quickest way to see whether PPO improved on the NNLS warm-start or regressed.
